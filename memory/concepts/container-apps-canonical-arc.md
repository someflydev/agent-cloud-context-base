# Container Apps Canonical Arc

PROMPT_29 completes the first-class Azure Container Apps canonical surface with four examples: `examples/canonical-container-apps/public-api-private-worker-jobs/python-fastapi/`, `examples/canonical-container-apps/public-api-private-worker-jobs/dotnet-aspnet/`, and `examples/canonical-container-apps/public-api-private-worker-jobs/typescript-hono/` for the public API, internal worker, Service Bus-triggered Container Apps Job, Cosmos DB, Blob Storage, Key Vault, managed identity, and KEDA queue-depth pattern; plus `examples/canonical-container-apps/dapr-pubsub-binding/dotnet-aspnet/` for two .NET 8 Container Apps using Dapr pub/sub, state, and secret components over Service Bus, Cosmos DB, and Key Vault. These examples establish the Azure `miniblue` Lane A local-provider pattern for managed containers and the isolated Azure Lane B shape. The Dapr example remains a tradeoff pattern for cross-language eventing and shared building blocks, not a default for single-component apps. PROMPT_30 completes the Kubernetes arc.
