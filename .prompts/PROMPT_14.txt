PART 1 — MANIFEST FILE FORMAT (schema_version: 2)
================================================================================

Every manifest in `manifests/*.yaml` follows this shape (derived from the
acb manifest schema, extended with cloud-specific fields):

```yaml
schema_version: 2
name: <manifest-name>
description: <one paragraph>
archetype: <archetype filename without .md>
primary_stack: <stack filename without extension | none for comparative reference manifests>
secondary_stacks:
  - <stack name>
  - <stack name>
provider: <aws | gcp | azure | multi>
runtime_tier: <function | managed_container | k8s | null>
primary_language: <python | typescript | go | java | dotnet | hcl | null>
generated_repo_target: <true | false>
iac_tool: <terraform | pulumi-typescript | pulumi-python | pulumi-go | pulumi-dotnet>
triggers:
  - <user-facing phrase>
aliases:
  - <user-facing phrase>
scenario_patterns:
  - <pattern id from context/scenarios/scenario-profile-map.yaml>
required_context:
  - context/doctrine/<file>.md
  - context/stacks/<file>.md
  - context/archetypes/<file>.md
  - context/scenarios/<file>.md
optional_context:
  - context/doctrine/<file>.md
  - context/workflows/<file>.md
preferred_examples:
  - examples/canonical-<family>/<example>/
recommended_templates:
  - templates/agent-md/AGENT.template.md
  - templates/claude-md/CLAUDE.template.md
  - templates/iac/<...>/<...>
  - templates/<runtime-tier>/<...>/<...>
repo_signals:
  - <file or path glob>
task_hints:
  - <workflow-name>
warnings:
  - <text>
bootstrap_defaults:
  - include_iac_dev_test_pair
  - include_smoke_tests
  - include_integration_tests
  - include_secret_binding_example
  - include_observability_bundle
iac_layout:
  root_dir: <iac/terraform | iac/pulumi/<language>>
  dev_path: <relative dir or file path for dev>
  test_path: <relative dir or file path for test>
  state_backend: <s3 | gcs | azurerm | pulumi-cloud | pulumi-self-managed>
env_isolation:
  env_var_prefix_dev: <PREFIX_DEV_>
  env_var_prefix_test: <PREFIX_TEST_>
  secret_path_dev: <provider-specific path>
  secret_path_test: <provider-specific path>
cloud_integration_test_strategy: <localstack | emulator | azurite | ephemeral_real | record_replay>
cost_band_per_test_run: <currency_amount_or_range>
smoke_test_expectations:
  - <expectation>
prompt_first_support: supported
```

For Terraform manifests, `iac_layout.dev_path` and `iac_layout.test_path`
point to the per-environment directories under `iac/terraform/`. For Pulumi
manifests, they point to the flat stack files under
`iac/pulumi/<language>/Pulumi.dev.yaml` and `Pulumi.test.yaml`.

`generated_repo_target: true` is the normal case for single-provider,
runnable repo profiles. Set `generated_repo_target: false` only for catalog /
reference manifests that intentionally do NOT map to a v1
`new_cloud_repo.py` scaffold. Those reference-only manifests MAY use
`runtime_tier: null`, `provider: multi`, `primary_stack: none`, or
`primary_language: null` where appropriate.
