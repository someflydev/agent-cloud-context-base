PART 3 — scripts/validate_iac_isolation.py (CLOUD-SPECIFIC GATE)
================================================================================

CLI: `python3 scripts/validate_iac_isolation.py <iac-root-dir>`

This is the gate that turns `iac-dev-test-isolation.md` doctrine into a
machine-checkable rule. Operates on either a Terraform tree (`iac/terraform/
{dev,test}/`) or a Pulumi tree using the canonical flat stack-file layout
(`iac/pulumi/<lang>/Pulumi.{dev,test}.yaml`). For imported repos, the
validator MAY tolerate legacy explicit env subdirs, but generated repos in
this arc use the flat Pulumi layout.

Checks:

  3.1  Terraform: both `dev/` and `test/` subtrees exist. Missing test
       tree fails; missing dev tree warns. Pulumi: either both flat stack
       files (`Pulumi.dev.yaml` and `Pulumi.test.yaml`) exist at the Pulumi
       root, or the repo is flagged as a legacy-imported layout.

  3.2  Terraform: `dev/backend.tf` and `test/backend.tf` exist with
       backend blocks; their `key` (S3) / `prefix` (GCS) / `key`
       (azurerm) values are different. Bucket / container / table
       names may match (shared backend infrastructure is fine), but
       the per-state key MUST differ.

  3.3  Pulumi: `Pulumi.dev.yaml` / `Pulumi.test.yaml` exist at the Pulumi
       root; `backend` config (when present) uses distinct paths; stack
       names are `dev` and `test` (or operator-declared equivalents in a
       comment); never the same name.

  3.4  Resource-name suffix discipline: scan `*.tf` / `*.ts` / `*.py` /
       `*.go` / `*.cs` for resource declarations. Each resource that
       includes `var.environment` / `pulumi.getStack()` /
       `Pulumi.Deployment.Instance.StackName` etc. is presumed
       env-suffixed. Resources without this signal are flagged as
       potential cross-env collisions.

  3.5  Secret-path disjointness: scan for `data.aws_secretsmanager_secret`
       / `aws_ssm_parameter` / `google_secret_manager_secret_version` /
       `azurerm_key_vault_secret` references. Names referenced from dev
       MUST NOT appear in test, and vice versa.

  3.6  No literal secret values: scan for `password = "..."`,
       `secret_key = "..."`, AKIA-prefixed strings, GitHub tokens
       (`ghp_*`, `github_pat_*`), `BEGIN PRIVATE KEY`, etc. Any hit
       fails.

  3.7  No `terraform.tfvars` or `pulumi.config.encrypted.yaml` checked
       in. Warn if present in the tree being scanned.

Output: a per-rule pass/fail table. Exit 0 clean, 1 violation, 2
environmental.

`--allow <rule-id>` lets an operator suppress a rule with a documented
reason (recorded in `.accb/validation/COVERAGE.json` deviations on next
`accb_payload` run).

PART 4 — scripts/verify_examples.py
================================================================================

CLI: `python3 scripts/verify_examples.py [--family <name>]`

Reads `examples/catalog.json` and `verification/example_registry.yaml`.
For each example:

  - confirm files declared in the registry exist
  - if a `verify_command` is declared, run it (timeout 5 minutes);
    success = exit 0
  - record `last_verified_at` and `last_verified_status` in the
    registry (only when `--update-registry` is passed; otherwise just
    print)

Concurrency: serial by default; `--parallel N` allows fan-out (only safe
for examples without shared external state).

Exit 0 clean, 1 any failure.
