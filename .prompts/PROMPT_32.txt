PART 1 — examples/canonical-integration-tests/
================================================================================

Five reference harnesses derived repos can copy and adapt. Each is a
small, self-contained directory with its own README, fixtures, and
runner script.

  1.1  aws-ministack-lambda-test/
       Python pytest harness using the ministack local provider bundle;
       aws-cli bootstrap of S3/DynamoDB/SQS resources; lambda invoke via
       ministack endpoint-url; teardown.

  1.2  gcp-minisky-firestore-pubsub-test/
       Python pytest harness using the minisky local provider bundle
       (Functions Framework plus Firestore, Pub/Sub, GCS, Secret Manager,
       Workflows / Tasks, BigQuery, Eventarc, and logs where covered);
       functions-framework run of the handler container; teardown.

  1.3  azure-miniblue-fn-test/
       .NET test using the miniblue local provider bundle; Azure
       Functions host / Core Tools-compatible runner; teardown.

  1.4  ephemeral-real-pulumi-up-down/
       Python harness using the Pulumi Automation API to apply a
       minimal `test` stack; runs assertions; tears down. Includes a
       max-runtime guard and an explicit "this lane costs real money"
       README.

  1.5  ephemeral-real-terraform-apply/
       Bash harness using `terraform apply -auto-approve` against a
       minimal test workspace; pytest assertions; `terraform destroy`
       teardown. Same cost guard.

Each harness must declare:
  - prerequisite environment variables
  - approximate runtime
  - approximate cost (Lane B and full/release harnesses)
  - skip behavior when prerequisites are missing

  - every example's structured-log shape declares the required fields
