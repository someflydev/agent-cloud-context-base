PART 1 — EXAMPLE LAYOUT CONTRACT
================================================================================

Each example is a self-contained directory with this structure:

```
examples/canonical-aws-lambda/<example-name>/
├── README.md                          # what the example does, run instructions
├── src/                               # handler code
├── iac/
│   ├── terraform/
│   │   ├── dev/{main.tf, backend.tf, dev.tfvars}
│   │   └── test/{main.tf, backend.tf, test.tfvars}
│   └── pulumi/<language>/
│       ├── Pulumi.yaml
│       ├── Pulumi.dev.yaml
│       ├── Pulumi.test.yaml
│       └── <entrypoint>               # __main__.py / index.ts / main.go
├── tests/
│   ├── smoke/                         # handler invocation + mocked SDK
│   ├── integration/
│   │   ├── lane-a-localstack/         # docker-compose.yml + pytest/jest
│   │   └── lane-b-ephemeral-real/     # pulumi/terraform up + tests + down
│   └── replay/                        # idempotency proof: same dedupe key twice
├── observability/
│   └── structured-log-shape.md        # JSON fields the handler emits
└── catalog-entry.json                 # appended into examples/catalog.json
```

Per-example registry entry (appended to
`verification/example_registry.yaml`):

```yaml
- family: canonical-aws-lambda
  name: <example-name>
  provider: aws
  runtime_tier: function
  language: <python|typescript|go>
  iac_tool_terraform: true
  iac_tool_pulumi: pulumi-<language>
  dev_test_disjoint: true
  verify_command: "bash examples/canonical-aws-lambda/<example-name>/tests/verify.sh"
  last_verified_at: null
  last_verified_status: pending
```

PART 3 — INTEGRATION TEST LANES (per example)
================================================================================

Lane A (LocalStack): a `tests/integration/lane-a-localstack/` directory
with:
  - `docker-compose.yml` running LocalStack with the services this
    example needs (`SERVICES=s3,dynamodb,sqs,sns,events,...`)
  - `bootstrap.sh` that creates required resources via aws CLI against
    `--endpoint-url http://localhost:4566`
  - `test_handler.py|.ts|.go` that invokes the handler against the
    LocalStack endpoints
  - `teardown.sh` that brings the stack down

Lane B (ephemeral real): a `tests/integration/lane-b-ephemeral-real/`
directory with:
  - `up.sh` that runs `pulumi up --stack test --yes` (or
    `terraform apply -auto-approve` against test workspace)
  - `test_handler.py|.ts|.go` that invokes the deployed function via
    the AWS API
  - `down.sh` that runs `pulumi destroy --stack test --yes`
  - a max-runtime guard (`timeout 1800` wrapping the suite)
  - explicit guidance: this lane requires AWS credentials and incurs
    real cost; the cost band is declared in the example README

Lane C (record/replay) is unused for AWS Lambda examples — no third-
party calls are mocked here. Bedrock-using examples would use Lane C
when added in a later arc.

PART 4 — REPLAY (IDEMPOTENCY) TESTS
================================================================================

Every example has `tests/replay/test_replay.py|.ts|.go`:
  - send the same trigger payload (same dedupe key) twice through the
    Lane A or Lane B harness
  - assert exactly-once effect at the persistence boundary (DynamoDB
    item count = 1; downstream events emitted = 1; etc.)
  - assert the second invocation returns successfully (no retry
    surprises) but performs no additional side effect

  function_name, dedupe_key, decision (ALLOW | FLAG | DROP, where

helper in Go. Observability emission must be visible in both Lane A
(LocalStack-captured logs) and Lane B (CloudWatch).
