# AWS Lambda Python

Load this stack for AWS Lambda functions written in Python. It owns handler shape, package layout, trigger contracts, idempotency, and provider bindings for Python 3.12 Lambda workloads generated by accb.

## Runtime Surface

- Runtime pin: Python 3.12 on AWS Lambda.
- Packaging: zip package by default; container image only when binary dependencies justify it; Lambda layers for shared dependencies.
- Dependency tool: uv preferred, pip acceptable when inherited from the repo.
- Cold-start budget: follow `context/doctrine/cold-start-and-runtime-selection.md`; avoid VPC attachment unless required.
- Limits: keep default timeout under 30s for request paths; declare any move toward the 15 minute Lambda maximum; choose memory from measured CPU and IO behavior.
- Supported triggers: S3, DynamoDB Streams, SQS, SNS, EventBridge rules, EventBridge Scheduler, API Gateway REST, API Gateway HTTP, API Gateway WebSocket, Cognito triggers, Kinesis, IoT, CloudWatch Logs subscriptions, CodePipeline events.

## Handler Skeleton

```python
from __future__ import annotations

from typing import Any


def handler(event: dict[str, Any], context: Any) -> dict[str, Any]:
    request_id = getattr(context, "aws_request_id", None)
    return {"ok": True, "request_id": request_id}
```

## Idempotency Pattern

- Derive the dedupe key from the trigger replay unit: S3 bucket/key/versionId, SQS messageId, SNS MessageId, EventBridge id, DynamoDB eventID, Kinesis eventID, API idempotency header, or Cognito userName plus triggerSource.
- Bind dedupe records to DynamoDB with environment, function name, dedupe key, status, effect checksum, and TTL.
- Keep idempotency outside process memory per `context/doctrine/serverless-state-discipline.md`.
- Replay test command: `uv run python -m pytest tests/integration/test_idempotency_replay.py`.

## Identity Binding

- Use one IAM role per function or tightly related function group; do not share broad execution roles across unrelated triggers.
- Scope policies to exact ARNs and actions needed by the trigger, dedupe table, secrets, logs, and downstream resources.
- Reference `context/stacks/identity-aws-iam.md` for the PROMPT_12 identity stack.

## Secrets

- Use AWS Secrets Manager for application secrets; inject names or ARNs through environment variables and fetch values through the SDK at runtime.
- Cache fetched secrets only within documented rotation tolerance.
- Default rotation cadence: 90 days unless the upstream credential requires shorter rotation.
- Reference `context/stacks/secrets-aws-secrets-manager.md`.

## Observability

- Emit structured JSON logs with `timestamp`, `level`, `service`, `env`, `aws_request_id`, `trigger_type`, `dedupe_key`, and `outcome`.
- Read trace context from Lambda context, API Gateway headers, SQS/SNS message attributes, or EventBridge trace headers when present.
- Use CloudWatch Embedded Metrics Format or AWS Lambda Powertools metrics for accepted, duplicate, retryable, and terminal events.
- Reference `context/stacks/observability-otel-cloud.md`.

## Anti-Patterns

- Hiding all trigger shapes behind an untyped `event` blob.
- Sharing mutable globals for state that must survive retries.
- Bundling large optional libraries into every function instead of isolating layers.
- Acknowledging stream or queue work before the durable effect is safe.
