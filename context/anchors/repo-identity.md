# Repo Identity Anchor

- Treat this repo as a context-routing system for cloud-native repos, not a product app.
- Use `accb` for the base and `.accb/` for generated repo payloads.
- Read routers, manifests, doctrine, and examples in that order of narrowing.
- Prefer repo signals over guesswork for provider, runtime tier, language, and IaC tool.
- Treat IaC dev/test isolation declarations as architectural signals.
- Keep generated cloud behavior aligned with canonical examples.
- Stop when more than one provider, runtime tier, or IaC tool is still plausible.
